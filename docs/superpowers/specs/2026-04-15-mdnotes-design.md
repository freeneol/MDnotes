# MDnotes 设计文档

轻量级跨平台 Markdown 阅读与编辑器。

## 概述

MDnotes 是一个轻便、简单的 Markdown 阅读和编辑工具，以阅读为主、编辑为辅。采用 Rust + iced 原生 GUI 框架构建，支持 Windows 和 Mac 系统，提供所见即所得（WYSIWYG）的编辑体验。

## 核心需求

- **定位：** 通用 Markdown 阅读器，编辑为辅助功能
- **编辑模式：** 所见即所得（WYSIWYG）
- **技术栈：** Rust + iced
- **Markdown 功能：** 基础 Markdown（标题、段落、粗体/斜体、链接、图片、列表、引用、代码块）
- **文件管理：** 单文件视图 + 侧边栏文件管理（主文件夹树形目录 + 最近文件）
- **分发方式：** 便携版（单文件）+ 安装包
- **平台：** Windows + Mac

## 架构

三层架构：

```
┌─────────────────────────────────┐
│          UI 层 (iced)           │
│   渲染视图 / 编辑视图 / 文件管理 │
├─────────────────────────────────┤
│         核心逻辑层               │
│   Markdown 解析 / 文档模型       │
├─────────────────────────────────┤
│         IO 层                   │
│   文件读写 / 配置持久化          │
└─────────────────────────────────┘
```

### 模块划分

- `main.rs` — 应用入口，iced Application 初始化
- `app.rs` — 应用状态机，消息处理
- `ui/` — UI 组件（工具栏、侧边栏、内容区、状态栏、欢迎页）
- `markdown/` — Markdown 解析与文档模型转换
- `io/` — 文件读写、配置管理

## UI 布局

上中下三段式布局，中间区域左右分栏：

```
┌─────────────────────────────────────────────┐
│  工具栏：[打开文件] [保存] [编辑/阅读]         │
├───────────┬─────────────────────────────────┤
│           │                                 │
│ 主文件夹   │                                 │
│ ├── notes/│       主内容区域                  │
│ │  ├── w… │  （WYSIWYG 渲染的 Markdown）      │
│ │  └── i… │                                 │
│ ├── diary…│                                 │
│ └── todo… │                                 │
│[打开文件夹]│                                 │
│           │                                 │
│ ────────  │                                 │
│ 最近文件   │                                 │
│ · readme… │                                 │
│ · meeting…│                                 │
│           │                                 │
├───────────┴─────────────────────────────────┤
│  状态栏：文件路径 | 字数统计 | 编辑状态        │
└─────────────────────────────────────────────┘
```

### 上部 — 工具栏（全宽）

- `[打开文件]` — 系统文件选择对话框
- `[保存]` — 保存当前文件（无文件打开时禁用）
- `[编辑/阅读]` — 切换模式（无文件打开时禁用）

### 中部 — 左右分栏

**左侧边栏**（固定宽度，可拖拽调整）：

- 主文件夹树形目录：显示用户设定的主文件夹下的 `.md` 文件和子目录结构。首次使用时显示 `[打开文件夹]` 引导用户选择主文件夹。
- 分隔线
- 最近文件列表：按时间倒序，上限 20 个。

**右侧主区域：**

- 无文件时显示欢迎空状态（提示"选择或打开一个文件开始"）
- 有文件时显示 WYSIWYG 渲染的 Markdown 内容

### 下部 — 状态栏（全宽）

显示文件路径、字数统计、编辑状态。

## 交互逻辑

### 模式切换

- **阅读模式（默认）：** 打开文件后显示渲染后的 Markdown，只读。
- **编辑模式：** 双击内容区域或点击工具栏"编辑"按钮切换，内容变为可编辑的 WYSIWYG 视图。

### 文件操作

- 左侧单击文件 → 右侧加载并显示（阅读模式）
- 切换文件时有未保存修改 → 弹出保存提示
- 侧边栏宽度可拖拽调整

### 快捷键


| 快捷键            | 功能        |
| -------------- | --------- |
| `Cmd/Ctrl + O` | 打开文件      |
| `Cmd/Ctrl + S` | 保存文件      |
| `Cmd/Ctrl + E` | 切换阅读/编辑模式 |
| `Cmd/Ctrl + \` | 显示/隐藏侧边栏  |


## 数据流

### Markdown 解析与渲染

```
.md 文件 → pulldown-cmark 解析 → AST（事件流）→ 内部文档模型 → iced 富文本渲染
```

### 文档模型

将 Markdown AST 转换为内部文档模型（一组带样式属性的块元素）：

**块元素类型：** 段落、标题(h1-h6)、列表项、引用块、代码块、图片、分隔线

**内联样式：** 粗体、斜体、行内代码、链接

### 文件操作流程

- **打开：** 读取文件 → 解析 → 渲染 → 加入最近文件列表
- **保存：** 文档模型 → 序列化为 Markdown 纯文本 → 写入文件
- **未保存检测：** 文档模型与上次保存的快照对比

### 配置持久化

- 配置文件位置：`dirs::config_dir()/mdnotes/config.json`
- 存储内容：主文件夹路径、最近文件列表、窗口尺寸/位置、侧边栏宽度

## 构建与分发


| 平台      | 便携版       | 安装包                        |
| ------- | --------- | -------------------------- |
| Mac     | 单个二进制文件   | `.dmg`（通过 `create-dmg` 生成） |
| Windows | 单个 `.exe` | `.msi`（通过 `cargo-wix` 生成）  |


- Rust 静态链接天然支持单文件可执行程序
- GitHub Actions 实现 CI/CD，自动为两个平台构建
- `cargo-bundle` 辅助生成 macOS `.app` 包
